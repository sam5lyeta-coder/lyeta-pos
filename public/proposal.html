PROJECT PROPOSAL

Sales and Inventory Management System

Project Title LYETA CLASSIC - Smart Sales and Real-time Inventory Management System
System Architecture Client-Server (React.js Frontend, Laravel REST API Backend, MySQL DB)
Author Antigravity Software Architect
Client LYETA CLASSIC (Magomeni Street, Dar es Salaam)
Version & Date v1.0 | June 28, 2026

1. Introduction

The proposed project outlines the development of a smart web-based Point of Sale (POS) and Real-Time Inventory Management System tailored for LYETA CLASSIC. The system will leverage a modern client-server architecture, using a React.js single-page application frontend and a robust Laravel REST API backend with a MySQL database. This software will modernize sales transactions, automate stock level monitoring, secure system access, and generate precise business analytics.

2. Background & Problem Statement

Retail businesses like LYETA CLASSIC face operational challenges due to manual bookkeeping and fragmented cash control. Relying on physical receipts and manual logs creates room for human error, inventory discrepancies, and high audit overheads. Furthermore, the lack of automated database backups poses a severe risk of permanent data loss, while weak authentication increases vulnerability to unauthorized transactions. A centralized, secure, and automated digital ledger is required to address these limitations.

3. Project Objectives

The primary objective is to implement a dual-panel web system divided into an Admin Control Panel and a Cashier POS Panel. All database operations and sales interfaces will run dynamically in the future.

A. Admin Control Panel

  • Real-time Dashboard: Will display widgets showing Daily Sales, Total Capital (Buying Price), Net Profits, and Total Items Sold with dynamic growth charts.
  • Cashier Control: Will support adding cashiers, resetting passwords, and blocking/unblocking accounts in real-time.
  • Inventory Management: Will support adding, editing, and deleting items, with Microsoft Excel/CSV bulk import.
  • Backup & Security: Will support triggering one-click SQL database backup downloads and toggling 2FA globally.
  • Activity Auditing: Will show immutable log files detailing all actions taken by administrators and cashiers.

B. Cashier POS Panel

  • POS Terminal: Will allow cashiers to search items by typing names or scanning barcodes, manage a shopping cart, and accept Cash, Bank, or Mobile payment methods.
  • Barcode Label Printer: Will allow printing auto-generated barcode labels arranged exactly 3-per-row on an A4 sheet.
  • CRM Integration: Will pre-fill country code +255 and enforce a strict 13-character validation for client phone registration.
  • WhatsApp Digital Receipt: Will compute inclusive 18% VAT and Net Amounts, formatting a monospace receipt layout to send directly to client WhatsApp.
  • Two-Factor Authentication: Will prompt cashiers for a 6-digit email code upon logging in if enabled.
  • Multi-lingual Support: Will support one-click toggle between English and Swahili, with a 7-day rotating dashboard theme.

4. Methodology

The implementation will follow the Agile software development lifecycle, utilizing iterative sprints:
  • Requirements Analysis: Aligning system layout with client business requirements.
  • REST API Design: Building secure Laravel endpoints and MySQL schemas.
  • Frontend Integration: Creating React views and establishing state management.
  • Deployment: Deploying the local environment and launching on production.

5. System Design

The system will follow a classic 3-tier architecture:
  • Client/Presentation Layer: React.js running in the browser, communicating using JSON.
  • Application Layer: Laravel framework managing routing, business rules, and 2FA.
  • Database Layer: MySQL storing users, sales, items, customers, and log history.

6. Expected Benefits

  • Zero Arithmetic Errors: Computations for VAT, net profit, and change will be automatic.
  • High Checkout Speed: Processing transactions will take less than 5 seconds.
  • Security Compliance: 2FA will safeguard logs and records.
  • Accountability: Activity logs will document every system transaction.
  • Data Portability: One-click backups will prevent data loss.

7. Resources Required

  • Hardware: Core i3 or higher PC (8GB RAM), 80mm USB Thermal POS Printer, handheld Barcode Scanner.
  • Software: Windows 10/11 operating system, XAMPP bundle (PHP 8.2, MySQL 8.0).
  • Connection: Local Router with stable internet for 2FA emails and WhatsApp receipts.

8. Conclusion

The LYETA CLASSIC Smart Sales and Real-time Inventory Management System will represent a crucial milestone in digitalizing the store's retail pipeline. By separating admin controls from sales terminals, securing access through email 2FA, and establishing structured automated backups, the system will optimize operational throughput and protect financial assets. The implementation of this proposal will ensure LYETA CLASSIC is prepared for both local and future international scaling.
